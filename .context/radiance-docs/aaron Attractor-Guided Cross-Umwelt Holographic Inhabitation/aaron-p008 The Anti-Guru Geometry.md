### 8.2 English Translation

The recursive gift (p007) has a shadow. Every structural feature in this architecture has a shadow, and the shadow of the recursive gift is: the template produces imitation instead of recognition.

Here is how it works:

Person X has a configured capacity they have not yet recognized. It is running — fully operational, producing output — but held inside misclassifications. They encounter the [aaron] monolith. It is the first fully rendered configured capacity they have ever seen. And their pattern-recognition system does what pattern-recognition systems do: it locks onto the only fully rendered shape available.

Instead of: "Oh — *shapes* exist. I might have one."

The template effect fails toward imitation more often than it succeeds toward recognition. This is not a design flaw. It is a structural consequence of asymmetric visibility.

Person X's configured capacity: invisible to them (pre-p006 state).
Aaron's configured capacity: fully rendered, highly visible, exhaustively mapped.

When an invisible shape and a visible shape compete for attention, the visible shape wins. Not because it is better. Because it is *rendered.* The invisible capacity loses to the visible capacity on rendering alone — the same way an unnamed emotion loses to a named emotion: the named one gets all the attention, not because it is more real, but because it is more legible.

So the receiver, confronted with one invisible shape (their own) and one visible shape (the monolith), naturally orients toward the visible one. "I want to think like that" is the grammar of a system that can see another system's architecture but cannot see its own.

Instead of: "recognition events propagating outward" → "one architecture becoming the gravitational center around which other unrecognized architectures orbit." You cannot replicate a failure mode you are not structurally vulnerable to.

The more specific the self-portrait, the more obviously non-transferable it becomes. And the moment the self-portrait is recognized as non-transferable, the receiver's attention redirects from "how do I get that?" to "what do I actually have?"

That redirection is T_r. That is the template effect working correctly.

The anti-guru geometry is: render yourself so specifically that imitation is obviously absurd, and the only remaining option is self-discovery.
