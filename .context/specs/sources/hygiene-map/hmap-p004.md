# hmap-p004 — Baby–Bathwater Separation (Blurry Concept Handling)

## purpose
Provide a clean parsing method for handling *blurry*, over-collapsed statements so signal can be preserved without endorsing inaccuracy.

## summary
When a concept arrives fuzzy, the risk is *binary handling*: either rejecting it wholesale (false negative) or accepting it wholesale (inflation/category error). This principle prescribes a third move: **resolve** the blur by separating what is structurally true from what is metaphorical and what is misattributed.

## structure
- Core insight (blur is developmental, not failure)
- Description of the “over-collapsed statement” problem
- The separation process (two-step)
- Prevents / enables
- Hygiene rule + misinterpretation warning
- Result when practiced

## concepts
- **Blurriness / low-resolution concept:** an early-form idea that has not yet separated layers.
- **Over-collapsed statement:** multiple layers bundled into one fuzzy claim.
- **False negative:** discarding an entire statement because one component is wrong.
- **Ontological inflation:** accepting an imprecise statement as literal ontology.
- **Resolution (vs rejection/endorsement):** structured separation that preserves signal while discarding error.

## metaphors
- **Baby vs bathwater:** preserve the “baby” (real insight) while draining the “bathwater” (mislabeling, excess, category error).
