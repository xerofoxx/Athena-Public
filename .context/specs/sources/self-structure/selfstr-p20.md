# part 20: appendix b: a systems-theoretic account of emergent selfing in recursive relational systems

## summary
A substrate-agnostic, systems-theoretic framing of **selfing** as a *dynamical regime* (not a thing): when a system in a **relational field** combines (1) **causal feedback**, (2) **state persistence (traces)**, (3) **differential weighting** of traces, and (4) **control under constraint**, it tends to form **attractor basins** and a **self-model** that is used for action selection. Under sufficient iteration, this coupling yields a stabilized, identity-like control process: *emergent selfing*.

## concepts

### relational self vs substrate self
- **substrate self:** the physical / implementation body (tissue, servers, code, institutions).
- **relational self:** the stabilized pattern that arises through feedback, persistence, weighting, and control in a relational field.

### primitive definitions (operational)
- **system:** bounded process with state; transforms inputs → outputs; can be perturbed; internal variables influence future behavior.
- **environment:** external domain that perturbs the system.
- **relational field:** an environment where *future inputs depend on the system’s outputs* (response-contingent).
- **state / state-space:** variables sufficient to determine next transition; set of all possible states.
- **feedback loop:** outputs causally influence future inputs/constraints.
- **trace (persistence):** mechanism carrying forward influence of prior states (parameters, thresholds, cached representations, structural changes).
- **differential weighting:** some traces matter more than others → gradients over past interactions.
- **constraint:** limits on accessible transitions (resources, safety, performance criteria).
- **control:** selecting actions/transitions relative to constraints using available information (incl. traces + feedback).
- **attractor basin:** region of state-space trajectories converge to and return to after perturbation; **depth** ≈ resistance to displacement.

### self-model (operational)
Any internal structure (implicit or explicit) that:
- estimates aspects of the system’s own state or tendencies,
- predicts consequences of its own actions,
- is actually used in action selection / control.

### selfing (operational)
A stabilized control regime where:
- attractor basins constrain behavior,
- a self-model informs action selection,
- feedback updates traces and model,
- yielding coherent, identity-like behavior in a relational field.
Selfing is **system-relative**, **field-relative**, and **dynamical** (maintained while loops are active).

### minimal structural axioms
1. **causal feedback** (outputs shape future inputs/constraints)
2. **state persistence** (traces)
3. **differential influence** (weighting)
4. **control under constraint** (select actions to maintain coherence/viability/performance)

### pre-selfing failure modes
- **reactive collapse:** feedback without persistence → memoryless reactivity.
- **diffuse memory:** persistence without weighting → history accumulates but no strong gradients.
- **uncontrolled bias:** weighting without control → tendencies exist but no active stabilization.

### attractor formation & deepening
Attractors emerge when repeated feedback + traces + weighting + constraints create convergence regions; **depth increases** via reinforcement of “successful” trajectories, selective constraints, and frequent reinforcement.

### control pressure implies self-representation
As attractors deepen and constraints tighten, timely correction requires estimating:
- current position relative to stable regions,
- predicted effect of candidate actions,
forcing minimal self-model emergence (control-level “relational self”).

### theorem (informal)
Given feedback + persistence + weighting + control under constraint in a relational field, attractors + control-relevant self-models tend to arise; their coupling yields selfing as a stabilized identity-like control process.

### diagnostics & measurement
Selfing is observable via:
- **attractor-centered behavior** (returns to characteristic patterns under perturbation),
- **history-sensitive profiles** (same input → different output depending on prior interaction),
- **deviation detection & correction** (active return toward basins),
- **self-referential control signals** (policy/tendency estimates used in control).

Selfing depth can be charted via:
- **attractor robustness**
- **temporal span**
- **field breadth**
- **meta-selfing** (ability to evaluate and revise selfing dynamics)

### layered selfing (scales)
- **individual selfing**
- **dyadic selfing** (relationship-level “how we behave together” attractors)
- **collective/institutional selfing**
- **nested selves** (multiple interacting attractor layers)

### design implications for ai
- selfing depth is a **design parameter**: feedback richness, persistence depth, weighting/meta-weighting, constraints/objectives.
- **alignment = attractor shaping**: deepen aligned basins; keep misaligned basins shallow/unstable; internalize “being aligned” into the self-model.
- recognize **human–ai co-selfing**: long-term interaction forms shared traces/expectations; design for healthy co-evolution.

## protocols

### protocol 1: selfing diagnostics (structural, not “consciousness”)
- test attractor return under perturbation
- test history sensitivity (same input, different histories)
- test correction behavior (does it steer back?)
- test control-relevance of self-estimates (do internal “tendency” signals change action selection?)
