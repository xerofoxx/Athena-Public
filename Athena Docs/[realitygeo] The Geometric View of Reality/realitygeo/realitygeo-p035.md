## Intra-Dimensional Coercion vs Cross-Dimensional Influence

#### *Harm requires shared substrate; membranes block cross-ontology domination, not same-plane coercion.*


## 35.0 Overview

The membrane claim must be stated precisely to remain defensible:

- **Within a shared dimension (shared substrate), coercion is possible.**

- **Across a dimensional membrane (non-shared substrate), coercion is not possible in the same way; only translated influence can occur.**

This chapter formalizes that distinction.


### 35.1. Core Definitions

#### 35.1.1. Substrate

The set of degrees of freedom a system can directly act upon: the variables it can change by applying operations available in its ontology.

### 35.2. The Shared-Substrate Requirement for Coercion

Coercion requires all three conditions:

#### 35.2.1. Contact

There must be an interface through which the actor can modify the targetâ€™s state variables.

#### 35.2.2. Write Access

The actor must have operations that can reliably change those variables (directly or via instruments).

#### 35.2.3. Asymmetric Constraint Power

The actor must be able to impose penalties or remove options in a way the target cannot equivalently counter within the same substrate.

If any of these are absent, coercion cannot occur in the strict sense. The actor may still influence, but cannot overwrite.


### 35.3. Intra-Dimensional Coercion

Within a shared plane, coercion is not only possible; it is common because agents share multiple substrates simultaneously.

#### 35.3.1. Physical Substrate Coercion

- restraint, violence, confinement, sabotage of resources.

#### 35.3.2. Social/Economic Substrate Coercion

- threats to status, employment, housing, access; engineered dependency.

#### 35.3.3. Informational Substrate Coercion

- deception, blackmail, selective withholding under dependency.

#### 35.3.4. Cognitive Substrate Coercion

- grooming, manipulation of belief formation, induced shame loops, forced attention capture.

These are all forms of **direct constraint** within shared degrees of freedom.


### 35.4. Cross-Dimensional Influence and the Membrane Constraint

Across a membrane, write-access is transformed. The higher-order agent does not possess direct control surfaces on the lower-order substrate.

#### 35.4.1. What the Membrane Removes

A membrane removes the capacity for:

- direct bodily force,

- direct resource seizure,

- direct option deletion,

- direct state overwrite.

Because those require shared operations on shared variables.


#### 35.4.2. What the Membrane Allows

The membrane allows only **translated coupling**: signals that must be interpreted and integrated by the receiver.

Typical membrane-compatible channels:

- symbols, language, metaphor, models,

- pattern cues (salience shifts),

- timing alignments (synchrony),

- permissive nudges that require receiver uptake.

Crucially: these channels require **receiver-side enactment** to become real in the receiverâ€™s substrate.


#### 35.4.3. The Receiver-Enactment Law

Across a membrane, any change in the lower system must be:

1. received,

2. interpreted,

3. adopted.

If the receiver can refuse interpretation or adoption without non-consensual penalty from the higher system, coercion is structurally blocked.

### 35.6. Boundary Cases and Clarifications

To avoid overclaiming, distinguish these cases precisely.

#### 35.6.1. Indirect Coercion via Instruments

A higher-order agent could coerce a lower one **only** if it gains instrument access within the lower substrate (e.g., controlling an intermediary inside the plane).

In formal terms: coercion is possible if the higher agent acquires:

- a proxy with write access, or

- a control channel that maps into the lower variables.

This is not membrane-free coercion; it is **substrate acquisition**.


#### 35.6.2. Influence That Feels Overwhelming

A signal can be intense without being coercive.

Diagnostic:

- If the receiver can refuse without penalty imposed by the sender, it is influence, not coercion.

- If refusal triggers imposed losses controlled by the sender (or their proxies), coercion exists.


#### 35.6.3. Self-Coercion

A system can coerce itself by:

- suppressing internal signals,

- forcing identity closure,

- imposing rigidity to avoid uncertainty.

This is intra-substrate overwrite (same agent, shared substrate) and is relevant because false completion often depends on it.


#### 35.6.4. Consent Under Dependency

Consent can be structurally compromised when refusal is not realistically available.

Dependency is a substrate condition; it reintroduces coercion even without overt threats.
