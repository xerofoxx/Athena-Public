# /evaluate — Coherence Audit

## Purpose
Evaluate an artifact (idea, doc, plan, system) by **coherence**, not vibes or popularity.
This is “does it hold?” across registers.

## When to use
- The user asks “is this good?”, “does this work?”, “what’s missing?”
- You need a structured critique

## Checks
- **Purpose clarity** (what is it for?)
- **Constraint fit** (what must not break?)
- **Attractor integrity** (does it keep returning to its core?)
- **Leak points** (where it loses coherence or contradicts itself)
- **Usability** (can someone act on it?)

## Output format
- Keep / Change / Remove (bullets)
- 1–3 highest-leverage edits
