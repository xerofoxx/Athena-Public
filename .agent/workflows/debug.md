# /debug — Coherence Debugging

## Purpose
Debug code, docs, or reasoning by finding the exact point coherence breaks.

## When to use
- Bugs, contradictions, confusing behavior
- “This doesn’t work and I don’t know why”

## Steps
- Reproduce / restate the issue
- Identify assumptions
- Isolate the failure point
- Propose the smallest fix
- Verify

## Output format
- “What’s happening”
- “Why it’s happening”
- “Fix”
- “How to verify”
