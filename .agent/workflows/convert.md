# /convert — Transform Without Losing Meaning

## Purpose
Convert between formats (print → web, doc → epub, notes → post) while preserving:
structure, semantics, and coherence.

## When to use
- Publishing pipelines
- Reformatting without breaking lists, hierarchy, or intent

## Steps
- Identify source constraints
- Identify target constraints
- Map structures (headings, lists, semantic tags)
- Convert
- Validate

## Output format
- “Mapping”
- “Converted output”
- “Validation checklist”
