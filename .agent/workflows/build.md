# /build — System Assembly (Selfing-Aware)

## Purpose
Build systems (tools, processes, frameworks) with awareness of:
feedback, persistence, weighting, and control under constraint.
In other words: **how the system will start selfing** once deployed.

## When to use
- Designing an agent, workflow, community system, or product
- Any time “behavior over time” matters

## Steps
1. **Feedback topology**
2. **What persists (traces)**
3. **What gets weighted**
4. **Constraints + control knobs**
5. **Safety: prevent bad basins**
6. **Metrics: measure selfing depth**

## Output format
- A structured checklist
- A minimal “first implementation” plan
