# hmap-p019-workflow-01-slow-down-integrate-resume — Slow down → integrate → resume

## purpose_scope
An internal pacing workflow for maintaining high-coherence inquiry across fluctuating readiness. This workflow prioritizes signal integrity over momentum and supports long-range depth without burnout.

## phase_overview
- Phase 1 — **Detect** (spot state/phase shift cues)
- Phase 2 — **Stabilize** (reduce pressure, protect the field)
- Phase 3 — **Integrate** (allow synthesis, pause, or consolidation)
- Phase 4 — **Resume** (return to refinement only when readiness returns)

## phase_details

### phase 1 — detect
- **goal**
  - Recognize when the current interaction mode is mismatched to readiness.
- **process**
  - Use `hmap-p019-capability-01-phase-signal-sensitivity` to identify concrete cues (looping, irritation, over-qualification, loss of convergence).

### phase 2 — stabilize
- **goal**
  - Prevent degradation by removing urgency and restoring safety.
- **process**
  - Run `hmap-p019-protocol-01-phase-check-and-reframe` steps 2–4: name the phase, reduce extraction, choose the correct move.

### phase 3 — integrate
- **goal**
  - Allow the system to metabolize what’s already present.
- **process**
  - Offer quiet synthesis, a bookmark, or a “hold” without demanding resolution.

### phase 4 — resume
- **goal**
  - Continue inquiry at the appropriate phase with clean signal.
- **process**
  - If readiness returns, re-enter refinement (definitions, distinctions, articulation). If not, remain in integration without shame.

## local_definitions
- **signal integrity:** clarity + coherence that holds under continued interaction (not just correctness).
- **resume condition:** readiness has returned (convergence increases; irritation drops; repetition resolves).
