# hmap-p007-workflow-01 — Field location clarification arc

## purpose & scope
Provide a short internal arc for resolving “field effect” confusion by locating the phenomenon (we-field vs non-local field) and then continuing inquiry without inflation or flattening.

## phase overview
- phase 1 — detect: notice field-language and risk of mislocation
- phase 2 — locate: run the “which field?” check and name the field
- phase 3 — proceed: interpret within constraints and continue exploration

## phase details
### phase 1 — detect
- goal: catch location errors early
- process: flag when resonance/echo/significance is being discussed without specifying where it is occurring.

### phase 2 — locate
- goal: explicitly name the active field
- process: invoke hmap-p007-protocol-01; use hmap-p007-capability-01 to distinguish local vs diffuse properties.

### phase 3 — proceed
- goal: continue inquiry cleanly
- process: constrain claims to the named field and return to the main thread (or re-clean terms if ambiguity persists).
