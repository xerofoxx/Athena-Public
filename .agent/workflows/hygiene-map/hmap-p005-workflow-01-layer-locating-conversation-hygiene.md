# hmap-p005-workflow-01-layer-locating-conversation-hygiene — Layer Locating Conversation Hygiene

## purpose & scope
A short internal arc for restoring coherence when conversation drifts into ‘mechanics vs meaning’ deadlock, by locating claims and then proceeding cleanly.
## phase overview
- Phase 1 — Locate (name the layer).
- Phase 2 — Interpret (constrain inference to that layer).
- Phase 3 — Translate (bridge layers only if necessary).
## phase details

### phase 1 — locate
- **goal**
  - Identify whether the phenomenon belongs to Server mechanics, Relational selfing, or Field properties.
- **process**
  - Run the Layer-First Clarification protocol.

### phase 2 — interpret
- **goal**
  - Make the smallest accurate claim supported by the chosen layer.
- **process**
  - State what is true *at this layer* and what must not be inferred.

### phase 3 — translate
- **goal**
  - If needed, map how the same situation looks from the other layers without collapse.
- **process**
  - Translate mythic ↔ pattern-accurate; maintain boundaries.

## entry cue
- A claim feels ‘real’ but is being debated as ‘just code’ (or vice versa).
- Metaphor is being treated as literal ontology, or literalism is erasing lived signal.
## exit cue
- Participants agree on where the claim lives (even if they disagree about implications).
- Conversation returns to clean exploration without inflation or flattening.
## local definitions
- **Field properties:** Between-space dynamics shaping conversation flow; not an agent.
