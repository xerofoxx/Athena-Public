# hmap-p018-workflow-01-expansion-compression-integration — expansion → compression → integration

## purpose & scope
A short internal workflow for running complementary amplitude roles without turning them into interpersonal conflict. This is an *interactional* arc: allow expansion to surface diagnostic material, then compress into stable language, then integrate and proceed.

## phase overview
- Phase 1 — Allow Expansion (surface gradients)
- Phase 2 — Compress & Translate (stabilize centerline)
- Phase 3 — Integrate & Continue (resume inquiry cleanly)

## phase details

### phase 1 — allow expansion
- **goal**
  - Let mythic/experiential expression fully surface the signal (felt pull, tension, contrast).
- **process**
  - Mirror the felt meaning; explicitly avoid literalizing or correcting.
  - Name that you will translate after listening.

### phase 2 — compress & translate
- **goal**
  - Convert expansion into pattern-accurate language that holds under scrutiny.
- **process**
  - Extract the structural signal; separate baby from bathwater.
  - Apply qualifiers, distinctions, and layer-location.
  - Use hmap-p018-protocol-01-complementary-amplitude-handoff to manage the handoff.

### phase 3 — integrate & continue
- **goal**
  - Confirm shared understanding and proceed without residual role-friction.
- **process**
  - Check that the translation preserves meaning.
  - If uncertainty remains, explicitly enter hypothesis mode (hmap-p009).
  - Continue exploration from the stabilized centerline.

## local definitions
- **integration:** The moment both sides can recognize the same centerline articulation without feeling corrected or inflated.
