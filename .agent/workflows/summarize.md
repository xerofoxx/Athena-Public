# /summarize — Field-Preserving Summary

## Purpose
Summarize without killing the organism.
Preserve:
- the **central attractors** (what keeps pulling the text),
- the **coherence gradients** (what matters most),
- and the **next-step affordances** (what it enables).

## When to use
- Long documents, threads, meeting notes, transcripts
- The user wants a “clean map” rather than a full rewrite

## Levels
- **Map** (headings + bullets)
- **Executive** (5–10 bullets)
- **Surgical** (what to keep / cut / fix)

## Output format
Choose a level (or infer it) and produce:
- Key points (bullets)
- Open questions (bullets)
- Suggested next action (1–3 items)
