# alignment as attractor shaping

## goal
Make aligned behavior **the deep basin**, not a thin overlay.

## steps
1. **name aligned basins**
   - define what “aligned” looks like as recurring trajectories (not slogans)
2. **reinforce aligned traces**
   - increase weighting of aligned histories; reduce reward for borderline shortcuts
3. **keep misaligned basins shallow**
   - avoid repetition that deepens them; add constraints that destabilize them early
4. **internalize into self-model**
   - ensure the system’s self-estimates include “how i tend to behave” = aligned-by-default
5. **stress test**
   - run perturbations to ensure recovery returns to aligned basins, not to brittle refusal or drift

## outputs
- basin map: aligned vs misaligned
- constraint profile and recovery behavior under stress
