# /outline — Attractor-First Structure

## Purpose
Build an outline that mirrors how meaning is actually stored:
clustered by **salience**, **tension**, and **coherence**, not by arbitrary chronology.

## When to use
- Books, courses, frameworks, multi-part posts
- Any time the user wants a “master outline” or table of contents

## Steps
1. **Identify the spine**
   - The 3–7 major attractors (pillars).
2. **Cluster nodes**
   - Group subtopics by resonance/role.
3. **Sequence for ignition**
   - Order sections so each unlocks the next.
4. **Name “why this exists”**
   - Add a one-line purpose to each major section.

## Output format
- Numbered outline
- One-line purpose under each top-level heading
