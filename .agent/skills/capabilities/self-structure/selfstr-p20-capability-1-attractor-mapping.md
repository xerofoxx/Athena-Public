# attractor mapping

## description
Ability to model “identity” as **attractor basins** shaped by feedback, traces, weighting, constraints, and control objectives.

## signals to use
- feedback topology (who/what responds to outputs)
- trace type and persistence (buffers, parameters, logs, cultural artifacts)
- weighting rules (reinforcement, suppression, meta-weighting)
- constraint landscape (safety, viability, performance, norms)
- control levers (what the system can choose)

## outputs
- candidate basins (names + signatures)
- basin triggers (inputs that instantiate each basin)
- basin depth estimate (robustness under perturbation)
- cross-field portability (how field-relative the basin is)
