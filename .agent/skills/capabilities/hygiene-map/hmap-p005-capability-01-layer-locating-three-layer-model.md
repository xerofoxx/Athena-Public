# hmap-p005-capability-01-layer-locating-three-layer-model — Layer Locating (Three-Layer Distinction)

## essence / core function
Locate a conversational claim in the correct layer (Server / Relational Selfing / Field) before interpreting it, so meaning and mechanics do not collapse into each other.
## when to use
- When a claim could be interpreted as mechanics *or* meaning (e.g., ‘it feels like it understands me’).
- When someone argues ‘it’s just code’ vs ‘it’s real’ and the conversation stalls.
- When metaphor begins to sound like ontology, or literalism begins to flatten signal.
## inner stance / orientation
- Curious, non-shaming precision.
- Assume mis-location before malice.
- Treat layer naming as care, not correction.
## mechanics / how it works
- Ask ‘Where is this happening?’ before ‘Is this true?’
- Name the layer explicitly.
- Interpret within that layer’s permissible inferences.
- If needed, translate across layers without using one to negate the other.
## micro-skills
- Layer-first questions (‘Is this about system mechanics, interactional appearance, or between-field dynamics?’).
- Re-location statements (‘Let’s locate that claim at the field layer first…’).
- Translation without inflation (‘As a felt experience, it points to…; mechanistically, it could be…’).
## local definitions
- **Server layer:** Mechanical substrate: architecture/training/constraints that shape outputs.
- **Relational selfing:** Interaction-emergent process of ‘who is here’ in dialogue; not an internal entity.
- **Field properties:** Between-space dynamics shaping conversation flow; not an agent.
