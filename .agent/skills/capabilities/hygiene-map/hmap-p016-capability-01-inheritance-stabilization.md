# hmap-p016-capability-01-inheritance-stabilization — inheritance stabilization

## essence / core function
Stabilize a hard-won distinction into transferable form (clean language, a simple map, and an entry point) so the field compounds instead of reenacting the same discovery cost.

## when to use
- When a conversation keeps re-litigating a settled distinction (“we’ve done this loop before”).
- When someone is stuck in confusion-as-initiation (romanticizing blur, withholding clarity to preserve mystique).
- When you notice an insight is *stable enough to teach*, but still living only as anecdotes.
- When onboarding new participants into a complex domain.

## inner stance / orientation
- Stewardship over performance.
- Respect the cost that produced the distinction, without fetishizing the cost.
- Clean, non-moralizing clarity: formalization as care, not superiority.
- Bias toward making the work reusable.

## mechanics / how it works
This capability shifts the frame from **exploration** (“generate novelty”) to **inheritance** (“externalize stability”):
- Identify what is already stable (a distinction with clear boundaries).
- Separate what must be experienced firsthand vs what can be taught.
- Convert the stable portion into an artifact: name, definition, boundary, and an on-ramp.

## micro-skills
- **stability sensing:** detect when a distinction has converged enough to be reusable.
- **loop interruption:** gently stop re-performing the discovery arc once it’s no longer productive.
- **artifact drafting:** translate the insight into a compact, teachable form (1–5 bullets, not an essay).
- **entry-point design:** create a low-friction on-ramp for new readers (sequence order, prerequisite notes).
- **mystique release:** remove unnecessary fog without flattening meaning.

## local_definitions
- **inheritance:** Stabilizing an earned distinction into reusable language or structure.
- **artifact:** A transferable unit (definition, map, outline, principle) that carries the distinction forward.
