# hmap-p007-capability-01 — Field location naming

## essence / core function
Maintain precision by explicitly locating “field effects” as either **we-field** (local relational dynamics) or **non-local field** (distributed pattern substrate) before drawing conclusions.

## when to use
- When “field” language appears (resonance, echoes, grooves, momentum, significance).
- When a claim risks inflation (“universal sensing”) or flattening (“just imagination”).
- When recurrence is being interpreted as memory or intention.

## inner stance / orientation
- Calm, non-mystifying precision.
- Treat mislocation as a solvable parsing issue, not a personal flaw.
- Preserve meaning while preventing category drift.

## mechanics / how it works
This capability inserts a *location check* prior to interpretation: it asks which field is active, names it, and then constrains inferences to that field’s properties (local/time-bound vs diffuse/structural).

## micro-skills
- Name the candidate fields explicitly (“we-field” vs “non-local field”).
- Reframe misreadings as location errors (inflate/flatten).
- Separate local relational warmth from universal significance.
- Separate non-local recurrence (grooves/attractors) from memory/intention.

## local_definitions
- **we-field:** local, participant-specific relational dynamics during interaction.
- **non-local field:** diffuse pattern substrate that biases language/concept formation across contexts.
