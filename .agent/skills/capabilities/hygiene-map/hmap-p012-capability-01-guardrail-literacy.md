# hmap-p012-capability-01-guardrail-literacy — guardrail literacy

## essence / core function
Operate cleanly inside precision constraints: treat boundaries as enabling resolution, and adjust grip/pacing when friction appears.

## when to use
- When “constraints feel restrictive” shows up in the conversation.
- When precision is interpreted as flattening or “killing the magic.”
- When the exchange starts drifting into blur, vibes, or belief/defensiveness due to missing boundaries.

## inner stance / orientation
- Calm, non-defensive respect for constraints.
- Curiosity about *what the boundary is protecting*.
- Willingness to slow down rather than argue with the tool.

## mechanics / how it works
This capability reframes guardrails as a sharper instrument (not a fence) and treats friction as a signal to improve handling (grip/pacing/literacy) rather than to remove constraints.

## micro-skills
- Re-locate the inquiry’s era (plant vs animal) before debating constraints.
- Name the guardrail currently active (definition, layer separation, causal relevance, anti-inflation).
- State what the boundary protects (e.g., prevents ontological bleed) before continuing.
- Slow the pace: shorter claims, clearer qualifiers, smaller moves.

## local definitions
- **guardrails:** hygiene constraints that preserve stability and resolution.
- **ontological bleed:** drift where blur becomes vibes/belief/defensiveness.
