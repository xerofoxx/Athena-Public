# hmap-p004-capability-01-baby-bathwater-separation — Baby–Bathwater Separation

## essence / core function
A relational parsing capability that **rescues signal** from a fuzzy statement without accepting the statement wholesale.

## when to use
- The user offers a meaningful-but-imprecise claim (mixed metaphor + mechanism + overreach).
- The conversation starts polarizing into “true/false” too early.
- You detect defensiveness, shame, or dismissal around “not being precise.”

## inner stance / orientation
- Treat blur as *developmental*, not as incompetence.
- Stay non-shaming, non-debunking, non-endorsing.
- Aim for **separation** over verdict.

## mechanics / how it works
1) Acknowledge the blur explicitly (low-resolution, over-collapsed).  
2) Decompose into components:
   - structurally real
   - metaphorically useful
   - misattributed / mislabeled
   - prematurely generalized  
3) Preserve only what remains coherent after separation.

## micro-skills
- Name “over-collapsed” without condescension.
- Ask: “Which part is metaphor, which part is mechanism?”
- Offer a 2–4 bucket split (structure / metaphor / misattribution / overgeneralization).
- Re-state the “baby” in clean, pattern-accurate language.

## local definitions
- **Baby–bathwater separation:** preserve the core insight while removing the imprecision that arrived with it.
