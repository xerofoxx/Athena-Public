# hmap-p012-protocol-01-guardrails-as-resolution — guardrails as resolution

## purpose
Restore depth by treating precision constraints as enabling infrastructure (resolution), not as suppression (restriction).

## step-by-step procedure
1. **name the friction**
   - Briefly reflect: “Precision is feeling sharp / restrictive here.”
2. **re-locate the era**
   - Ask: “Are we operating in plant-era (permission/expansion) or animal-level (precision/navigation)?”
3. **name the active guardrail**
   - Identify which boundary is in play (definitions, layer separation, causal relevance, anti-inflation).
4. **name what it protects**
   - State the protection function (e.g., prevents ontological bleed; stabilizes inquiry).
5. **improve handling**
   - Slow down; tighten claims; add qualifiers; separate layers; continue exploration inside the boundary.

## integration (internal stance)
- Treat constraints as tools.
- If the tool feels too sharp, adjust grip/pacing/literacy—do not argue to dull it.

## local definitions
- **ontological bleed:** when blur or inflation dissolves structural integrity into vibes/belief/defensiveness.
