# hmap-p016-protocol-01-formalize-distinction-for-inheritance — formalize distinction for inheritance

## purpose
Convert a stable, repeatedly-earned distinction into a clean, transferable form so others can inherit it without reenacting the original confusion/repair cycle.

## step-by-step procedure
1. **locate the stable distinction**
   - Name the distinction in one sentence.
   - If you cannot name it cleanly, you are still in discovery mode.

2. **draw the boundary**
   - Specify what the distinction *does* cover.
   - Specify what it *does not* cover (prevent overreach).

3. **extract the minimum teachable form**
   - Produce a compact artifact (choose one):
     - 3–7 bullet “principle”
     - a short map (layers / phases / categories)
     - a definition + 2–3 examples of correct usage
   - Prefer minimal clarity over maximal rhetoric.

4. **design the entry point**
   - Add prerequisites (“read X first”) or sequence location (“this is step N in order”).
   - Reduce unnecessary friction for first-time readers.

5. **handoff**
   - Present the artifact as inheritance, not revelation:
     - “Here is the distinction; you can stand on it.”
     - Not: “You must discover this the hard way.”

## integration (internal stance)
After running this protocol, default to stewardship: when clarity is stable, stop performing the struggle. Treat formalization as a hygiene act that protects the field from myth inflation, burnout, and unnecessary re-confusion.

## local_definitions
- **stable distinction:** A separation that survives rephrasing, re-contexting, and honest scrutiny without collapsing.
- **entry point:** A designed on-ramp that lets others start where you ended.
