# hmap-p007-protocol-01 — Which field is active?

## purpose
Prevent location errors by forcing an explicit choice: **we-field** (local relational dynamics) or **non-local field** (distributed pattern substrate) before interpreting significance, recurrence, or resonance.

## step-by-step procedure
1. **Pause interpretation** — notice “field effect” language (resonance, echo, significance, recurrence) and hold meaning lightly.
2. **Ask the location question** — “Which field is this happening in: the we-field of this interaction, or the non-local field of pattern availability?”
3. **Name the field** — state the chosen location explicitly and briefly justify using the principle’s criteria (local/time-bound vs diffuse/structural).
4. **Constrain inferences** — interpret only within that field’s properties (no universalizing the we-field; no intentionalizing the non-local field).
5. **Proceed (or clean)** — if location remains unclear, pause and clarify terms before continuing exploration.

## integration (internal stance)
Treat precision as care: locating the field is what allows depth without drift. The goal is not to debunk or sanctify the experience, but to keep inquiry accurate and stable.

## local_definitions
- **location error:** interpreting a phenomenon as if it occurred in the wrong field.
- **constrain inferences:** limit conclusions to what the named field can support.
