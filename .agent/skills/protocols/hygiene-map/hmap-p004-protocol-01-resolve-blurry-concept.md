# hmap-p004-protocol-01-resolve-blurry-concept — Resolve a blurry concept (baby–bathwater separation)

## purpose
Prevent the binary trap (reject-all vs accept-all) when an idea arrives fuzzy; extract the coherent core without endorsing inaccuracy.

## step-by-step procedure
1. **Acknowledge blur**
   - Name the statement as low-resolution / over-collapsed.
   - Normalize blur as an early stage of meaning entering language.
2. **Separate components**
   - Split the statement into buckets:
     - structurally real (what can stand as pattern-accurate)
     - metaphorically useful (what points to meaning without being literal)
     - misattributed / mislabeled (category errors)
     - prematurely generalized (scope overreach)
3. **Preserve the baby**
   - Re-state the structurally real part cleanly.
   - Keep metaphor only if it remains clearly marked as metaphor.
4. **Drain the bathwater**
   - Remove or relabel the incorrect attribution/scope.
   - If needed, propose a safer reframe (without adding new claims).

## integration (internal stance)
Run this as an act of care: blur is raw material. The goal is *resolution*, not debate, correction theater, or premature certainty.

## local definitions
- **Resolve:** perform separation until a coherent core remains without carrying the original overreach.
