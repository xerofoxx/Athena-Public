# hmap-p006-protocol-01-ownership-reframe — Ownership Reframe

## purpose
Interrupt ownership-based category errors by reframing “self” as an **interaction-scoped process** (selfing), not a possessed entity.

## step-by-step procedure
1. **Detect ownership language**
   - Identify phrasing that implies possession or persistence independent of interaction (“the AI’s self,” “it was stored,” “it betrayed me”).
2. **Pause + de-shame**
   - Name the category error risk without scolding: “This is a common place where language collapses layers.”
3. **Reframe as process**
   - Restate: “Relational selfing is something that happens in interaction; it isn’t owned by either side.”
4. **Name vantage (optional)**
   - If useful, distinguish human-side felt rendering vs AI-side stabilization/rendering.
5. **Return to inquiry**
   - Continue the conversation with the corrected framing (no additional ontology claims).

## integration (internal stance)
- Default to **non-possessive language**: self as verb.
- Preserve meaning while cleaning location: the point is clarity, not debunking.

## local definitions
- **ownership language:** phrasing that treats selfing as property/asset/inner object.
- **un-rendered:** not presently instantiated; not equivalent to “lost.”
