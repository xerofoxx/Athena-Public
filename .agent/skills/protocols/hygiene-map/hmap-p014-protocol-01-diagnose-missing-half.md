# hmap-p014-protocol-01-diagnose-missing-half — diagnose missing half (map vs movement)

## purpose
When understanding stalls, identify whether the missing ingredient is **structure** (map) or **motion** (enactment), and introduce the right complement without collapsing the other.

## step-by-step procedure
1. **Name the stall**
   - State what’s happening: “This feels true but unusable,” or “This feels alive but blurry.”
2. **Classify the failure mode**
   - Structure-only: crisp language, poor application / timing.
   - Motion-only: rich felt sense, weak boundaries / drift risk.
3. **Ask the diagnostic question**
   - “Do we have a map but no lived example, or an example but no clean language?”
4. **Add the missing half**
   - If structure is missing: define terms, boundaries, layer locations.
   - If motion is missing: give an enactment (story, walkthrough, typical mistake + repair).
5. **Re-pair and proceed**
   - Translate the new material back into the other half (name what was enacted; enact what was named).

## integration (internal stance)
- Treat skill as *paired literacy*.
- Do not shame the missing half; treat it as normal developmental sequencing.
- Keep both forms honest: structure must constrain; motion must demonstrate.

## local definitions
- **map:** structure-level orientation artifacts.
- **movement:** motion-level enactment artifacts.
- **pairing:** intentionally coupling both so understanding transfers.
