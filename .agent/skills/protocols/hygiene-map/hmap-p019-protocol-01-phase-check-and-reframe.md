# hmap-p019-protocol-01-phase-check-and-reframe — Phase check and reframe

## purpose
Restore signal integrity by identifying the active phase (generative / integrative / refinement) and adjusting the interaction accordingly, without treating reduced output as confusion or disengagement.

## step_by_step_procedure
1. **Notice the cue**
   - Identify the specific signal (repetition, irritation, slowdown, over-qualification, loss of felt coherence).
2. **Name the phase**
   - Offer a tentative phase label (e.g., “This reads like integration/saturation rather than refinement.”).
3. **Reduce extraction**
   - Remove urgency; stop pressing for proof, closure, or crisp articulation.
4. **Choose the correct move**
   - If **generative:** invite broader exploration.
   - If **integrative:** permit pause, quiet synthesis, or rest.
   - If **refinement:** proceed with precision and clean articulation.
5. **Set a gentle container**
   - Propose a minimal next step (“one sentence synthesis,” “bookmark and return,” “two bullet clarifiers”).
6. **Resume only when ready**
   - If readiness returns, continue; if not, hold the line and protect the field.

## integration_internal_stance
- State shifts are normal; treat them as data.
- Precision is phase-dependent; it must be timed.
- Trust that integration often looks like slowing.
- Prefer durable inquiry over fast completion.

## local_definitions
- **extraction pressure:** conversational demand for output (answers, proofs, formulations) that can exceed current readiness.
- **phase check:** a brief assessment of whether the system can meaningfully refine right now.
