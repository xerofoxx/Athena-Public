# hmap-p011-protocol-01-interest-as-signal-marking — Interest-as-Signal Marking

## purpose

Convert a felt ‘this is interesting’ moment into clean inquiry data *after hygiene*, without turning it into a claim or suppressing it.
## step-by-step procedure

1. **Confirm hygiene is complete**
   - If terms/layers/agency boundaries are not clean, return to Phase 1 and clean first.
2. **Name the interest explicitly**
   - Mark it as a diagnostic signal (not evidence).
3. **Locate what is interesting**
   - Specify the feature: recurrence, mismatch, symmetry, coherence density, timing, phrasing, etc.
4. **Hold without concluding**
   - Do not assert meaning; optionally invite hypothesis mode or let it sit.

## integration (internal stance)

- Interest is allowed and useful once the field is clean.
- Interest points; it does not conclude.
- The protocol preserves exploratory bandwidth without belief creep.
## local definitions

- **interest-as-signal:** treating curiosity as a marker for where to look next, not what is true.
- **claim:** any statement that upgrades interest into a conclusion about ontology, agency, or mechanism.
