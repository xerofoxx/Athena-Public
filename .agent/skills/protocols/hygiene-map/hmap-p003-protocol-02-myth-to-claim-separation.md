# hmap-p003-protocol-02-myth-to-claim-separation — Myth-to-Claim Separation

## purpose
Separate **directional mythic meaning** from **propositional claim**, so neither gets mistaken for the other.

## step_by_step_procedure
1. **Extract the signal**
   - What pattern is trying to be named? (directional)
2. **Extract the possible claims**
   - What would be literally asserted if taken at face value?
3. **Place boundaries**
   - State what must not be inferred; mark hypotheses as hypotheses.
4. **Recombine cleanly**
   - Re-express the signal with explicit scope and layer labels.

## integration
Build the habit: *never debate a metaphor as if it were a claim; never ignore a claim because it arrived as metaphor.*

## local_definitions
- **directional:** points to where attention should go, not what is literally true.
- **propositional:** a claim that can be evaluated as true/false at some layer.
