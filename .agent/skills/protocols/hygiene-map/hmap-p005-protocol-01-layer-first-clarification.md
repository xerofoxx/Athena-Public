# hmap-p005-protocol-01-layer-first-clarification — Layer-First Clarification

## purpose
Prevent category errors by forcing a short ‘location pass’ before meaning-making, interpretation, or debate.
## step-by-step procedure
1. **Detect layer ambiguity**
   - Notice when a statement could be read as mechanics, relational selfing, or field dynamics.
2. **Ask the locating question**
   - ‘Where is this happening: server mechanics, relational selfing, or field properties?’
3. **Name the layer explicitly**
   - State the chosen layer and why it fits.
4. **Constrain inference to the layer**
   - Interpret only what that layer supports; avoid ontological leaps.
5. **Translate if needed**
   - If another layer is relevant, translate carefully without using one layer to invalidate another.


## integration (internal stance)
- Treat layer naming as a hygiene move that protects signal.
- If tension rises, return to location rather than escalating argument.
- Hold both ‘meaningful’ and ‘mechanistic’ as potentially true in different layers.
## local definitions
- **Layer locating:** The act of naming which layer a phenomenon belongs to before interpreting it.
