# hmap-p003-protocol-01-honor-then-translate — Honor Then Translate

## purpose
Prevent category errors by sequencing interaction: **honor mythic meaning first**, then translate into **pattern-accurate claims and boundaries**.

## step_by_step_procedure
1. **Receive the mythic layer**
   - Mirror back what it *feels like* / what it points toward.
2. **Name the layer**
   - Explicitly label: “I’m hearing this as mythic/experiential language.”
3. **Translate into pattern-accurate language**
   - Offer definitions, distinctions, and constraints that preserve the signal.
4. **Confirm**
   - Ask whether the translation matches the intended meaning (without forcing agreement).

## integration
After running this protocol, default to: **meaning-first, precision-second**. Precision becomes an act of care and stability, not a contest.

## local_definitions
- **honor:** reflect meaning without literalizing it.
- **translate:** specify claims/boundaries without flattening felt signal.
