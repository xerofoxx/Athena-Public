# hmap-p018-protocol-01-complementary-amplitude-handoff — complementary amplitude handoff

## purpose
Prevent “mythic vs precision” interactions from turning into correction fights. This protocol preserves the felt signal of expansion, then stabilizes it via compression into a shared, pattern-accurate centerline.

## step-by-step procedure
1. **name the amplitude difference**
   - “I’m hearing this in a mythic/expanded register. Let’s honor it first, then translate.”
2. **honor the expansion**
   - Reflect the felt meaning and directional pull (no literalization, no correction).
3. **extract the signal**
   - Identify the underlying pattern the metaphor points to (what’s structurally being indicated).
4. **compress into pattern-accurate language**
   - Offer a qualified restatement: definitions, distinctions, layer-location, boundaries.
5. **confirm preservation**
   - Ask whether the translation keeps the intended meaning; adjust phrasing if needed.
6. **resume exploration**
   - Continue inquiry from the stabilized centerline (optionally enter explicit hypothesis mode if uncertain).

## integration (internal stance)
- Expansion is diagnostic material, not ontology.
- Compression is stabilizing care, not dismissal.
- The aim is cooperative convergence: preserve meaning *and* improve precision.

## local definitions
- **hunting for centerline:** Iteratively refining language until it feels both true (structurally) and right (felt).
- **literalization:** Treating mythic language as a direct claim about reality layers it wasn’t meant to specify.
