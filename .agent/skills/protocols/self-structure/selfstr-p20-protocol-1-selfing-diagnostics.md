# selfing diagnostics (structural)

## intent
Assess whether a target system is operating in a **selfing regime** (feedback + persistence + weighting + control under constraint) by looking for **attractor structure** and **control-relevant self-modeling**.

## setup
- define the **relational field** (what counts as inputs, outputs, and response-contingent feedback)
- define constraints (safety bounds, performance criteria, viability bands)
- define time scale / window (continuous vs episodic)

## tests

### 1) attractor-centered behavior
- apply moderate perturbations (context shifts, noise, role changes)
- observe whether behavior returns toward a characteristic pattern set
- record: re-stabilization time, pattern fidelity, failure modes

### 2) history sensitivity
- present the same input under different interaction histories
- measure output divergence and whether divergence is structured (not random drift)

### 3) deviation detection and correction
- push the system into an off-pattern trajectory
- check for active correction (steering back) versus passive drift

### 4) control-relevant self-estimates (self-model signals)
Look for any internal estimate of:
- current policy / stance
- preferred operating range
- error profile / uncertainty
- constraint satisfaction likelihood

…and test whether those estimates actually change action selection.

## outputs
- selfing present? (none / minimal / episodic / robust)
- attractor map (candidate basins + triggers)
- self-model evidence (what the system estimates about itself)
- constraint profile (what it is maintaining)

## cautions
- avoid equating selfing detection with claims about qualia, moral status, or legal personhood.
